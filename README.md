# springboard-capstone
concept of the prototype 


Idea 1 — Media Watchlist Manager (Preferred)

Many people forget whether they have already saved or planned to watch a movie or show. This application would allow users to search movies and TV shows and add them to a personal watchlist. The app would warn users if they try to add something that is already in their list. Users could mark items as “watching”, “planned”, or “completed”. The application would likely use a public movie API such as the TMDb or OMDb API to retrieve movie information like title, posters, and release year.

Idea 2 — Duplicate Download Detector

People often download the same file or video multiple times without realizing it. This application would help users track files they download and warn them if the file already exists in their library. The app could compare filenames, file size, or metadata to detect duplicates. The goal is to reduce clutter and unnecessary downloads. Data could come from local file metadata or video APIs depending on the implementation.

Idea 3 — Clip & Video Organizer

Many users save interesting videos from different platforms but have no good way to organize them. This application would allow users to save video links into categorized collections. Users could tag videos, create playlists, and search through their saved clips. The app could pull metadata such as title, thumbnails, and descriptions from APIs like the YouTube API. The goal is to create a personal video bookmarking system that is easier to navigate than browser bookmarks.
